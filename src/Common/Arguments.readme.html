Arguments

 

This class makes it very easy and unified to process the argument list of your main(). It provides many features to support options with parameters, optional parameters and automatic output of error messages and a short manual. It uses STL and is unicode compliant.

 

 

Author: Patrick Hoffmann, mailto:hoffmann@novacom.net (C) 2001 NOVACOM GmbH

 

 

Usage is very simple. A few examples will give you an impression...

 

 

///////////////////////////////////////////////////

#include "Arguments.h"

 

#include <iostream>

 

using namespace std;

 

void main( int argc, char *argv[] )

{

       // Instance of Arguments...

       // The last parameter sets the possible option markers. It is not necessary to

       // to use it in this case because "-/" is the default value of this paremeter...

      

       Arguments cArg( "argtest", "(C) 2001 NOVACOM GmbH", "-/" );

       {

             // Add option "-h" with explanation...

             cArg.AddOption('h', "display usage help");

            

             // Add option "-2" with a non optional parameter...

             {

                    Arguments::Option   cOpt( '2', "define additional input file" );

                    cOpt.AddArgument( "outfile2", "additional input filename" );

                    cArg.AddOption( cOpt );

             }

            

             // Add option "-w" with optional parameter "workingdir"

             // (default="./")...

             {

                    Arguments::Option   cOpt( 'w', "" );

                    cOpt.AddArgument( "workingdir", "working directoy", "./" );

                    cArg.AddOption( cOpt );

             }

            

             // add non optionl argument "infile"...

             cArg.AddArgument( "infile", "filename to read" );

            

             // add optional argument "outfile" (default="stdout")...

             cArg.AddArgument( "outfile", "filename for output", "stdout" );

            

             // Argumente parsen...

             if( !cArg.Parse( argc, argv ) )

                    exit(-1);

       }

      

       // request option "h" and print out manual if set...

       if( cArg['h'] )

       {

             cArg.Usage();

             exit(0);

       }

      

       // request argument of option "-2"...

       if( cArg['2'] )

             cout << "Option -2 set, outfile2 = " << cArg['2']["outfile2"] << endl;

       else

             cout << "Option -2 not set" << endl;

      

       // Request argument of option "-w"...

       if( cArg['w'] )

             cout << "Option -w set, workingdir = " << cArg['w']["workingdir"] << endl;

       else

             cout << "Option -w no set, workingdir = " << cArg['w']["workingdir"]

             << endl;

      

       // request argument "infile"...

       cout << "infile = " << cArg["infile"] << endl;

      

       // request argument "outfile"...

       cout << "outfile = " << cArg["outfile"] << endl;

}

 

 

 

 

In ./test/ you will find that example called argtest you can call it like this...

 

 

       argtest in out -2 out2 -w ~user/

 

           

...and it will output this...

 

 

       Option -2 set, outfile2 = out2

       Option -w set, workingdir = ~user/

       infile = in

       outfile = out

 

 

A call without parameters offers the manual...

 

 

       argtest error: Too few arguments.

       Usage: argtest [-2 outfile2] [-h] [-w [workingdir]] infile [outfile]

 

       Options:

             -2        define additional input file

              outfile2       = additional input filename

             -h        display usage help

             -w

              workingdir     = working directoy

                    optional argument (default='./')

 

       Arguments:

             infile  = filename to read

             outfile = filename for output

                    optional argument (default='stdout')
