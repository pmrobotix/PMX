25  /*
26  * edtaa3()
27  *
28  * Sweep-and-update Euclidean distance transform of an
29  * image. Positive pixels are treated as object pixels,
30  * zero or negative pixels are treated as background.
31  * An attempt is made to treat antialiased edges correctly.
32  * The input image must have pixels in the range [0,1],
33  * and the antialiased image should be a box-filter
34  * sampling of the ideal, crisp edge.
35  * If the antialias region is more than 1 pixel wide,
36  * the result from this transform will be inaccurate.
37  *
38  * By Stefan Gustavson (stefan.gustavson@gmail.com).
39  *
40  * Originally written in 1994, based on a verbal
41  * description of Per-Erik Danielsson's SSED8 algorithm
42  * as presented in the PhD dissertation of Ingemar
43  * Ragnemalm. This is Per-Erik Danielsson's scanline
44  * scheme from 1979 - I only implemented it in C.
45  *
46  * Updated in 2004 to treat border pixels correctly,
47  * and cleaned up the code to improve readability.
48  *
49  * Updated in 2009 to handle anti-aliased edges,
50  * as published in the article "Anti-aliased Euclidean
51  * distance transform" by Stefan Gustavson and Robin Strand,
52  * Pattern Recognition Letters 32 (2011) 252¨C257.
53  *
54  * Updated in 2011 to avoid a corner case causing an
55  * infinite loop for some input data.
56  *
57 */

77 /*
78  * A somewhat tricky function to approximate the distance to an edge in a
79  * certain pixel, with consideration to either the local gradient (gx,gy)
80  * or the direction to the pixel (dx,dy) and the pixel greyscale value a.
81  * The latter alternative, using (dx,dy), is the metric used by edtaa2().
82  * Using a local estimate of the edge gradient (gx,gy) yields much better
83  * accuracy at and near edges, and reduces the error even at distant pixels
84  * provided that the gradient direction is accurately estimated.
85  */
86 double edgedf(double gx, double gy, double a);
